# LetsBegin MCP Server

An MCP (Model Context Protocol) server that exposes LetsBegin project state as tools for AI agents.

## Quick Start (3 steps)

### 1. Clone and install (one time only)

```bash
git clone https://github.com/nickarino/LetsBegin.git && cd LetsBegin/mcp-server && npm install
```

### 2. Find your Supabase credentials

You need two values from the Supabase dashboard:

| Credential | Where to find it |
|---|---|
| **SUPABASE_URL** | Supabase dashboard → Settings → API → **Project URL** |
| **SUPABASE_SERVICE_KEY** | Supabase dashboard → Settings → API → **service_role key** (under "Project API keys") |

> **Note:** The `SUPABASE_URL` is also available as `NEXT_PUBLIC_SUPABASE_URL` in the web app's environment. If you're running the LetsBegin web app locally, you can copy it from `.env.local`.

> **Note:** The service_role key bypasses Row Level Security. You can use `SUPABASE_ANON_KEY` instead for read-only access.

### 3. Add to Claude Code

Add this to your Claude Code MCP configuration (`.claude/mcp.json` or project settings):

```json
{
  "mcpServers": {
    "letsbegin": {
      "command": "npx",
      "args": ["tsx", "/absolute/path/to/LetsBegin/mcp-server/src/index.ts"],
      "env": {
        "SUPABASE_URL": "https://your-project.supabase.co",
        "SUPABASE_SERVICE_KEY": "your-service-role-key"
      }
    }
  }
}
```

Replace `/absolute/path/to` with the actual path where you cloned the repo.

Then restart Claude Code and say: **"Plan my project: [your brief]"**

### Add to Cursor

In Cursor settings, add an MCP server with the same command and environment variables.

## Available Tools

| Tool | Description |
|------|-------------|
| `list_projects` | List all projects with titles, summaries, and progress |
| `get_current_task` | Get the highest-priority next task (optionally scoped to a project) |
| `get_project_context` | Get full project state: DAG, completed tasks, pending tasks |
| `mark_task_done` | Mark a task as completed, with optional notes |
| `add_task` | Add a new task to a project |
| `get_task_prompt` | Get the agent prompt for a task, ready to execute |
| `get_planning_prompt` | Get a planning prompt for a project brief — generate the plan with your own AI, then submit via `submit_plan` |
| `submit_plan` | Submit a plan JSON (with `project_title`, `summary`, and `nodes` DAG) generated by your own AI tool |

## Development

# Type-check
